Developer Documentation

Complete guide to integrating Kavach AI into your autonomous agent infrastructure for runtime security and threat protection.

Introduction #

Kavach AI is a zero-trust runtime firewall designed specifically for autonomous AI agents. It intercepts, analyzes, and secures every agent action before it reaches your infrastructure, providing real-time threat detection and policy enforcement.

What is Kavach AI?
Kavach AI acts as a security proxy between your AI agents and their target systems (databases, APIs, file systems). It analyzes each action for malicious intent, prompt injection attempts, and policy violations before allowing or blocking the request.

Key Features

  • Real-time Threat Detection — Analyzes agent actions in sub-2ms latency
  • Prompt Injection Protection — Detects and blocks injection attempts
  • Policy Engine — Enforce custom security policies per agent
  • Trust Scoring — Dynamic trust scores based on agent behavior
  • MCP Compatible — Works with Model Context Protocol servers
  • Enterprise Ready — Self-hosted with full data control

Components

  • Intent Engine — Analyzes the intent behind agent actions
  • Risk Engine — Calculates risk scores based on action context
  • Policy Engine — Enforces custom security policies
  • Trust Engine — Maintains dynamic trust scores for agents
  • Memory Inspector — Monitors agent memory for sensitive data

Security Pipeline #

Every agent action goes through a comprehensive security pipeline:

Security Pipeline Flow
Agent Action → Intent Analysis → Prompt Injection Scan → Risk Assessment → Policy Check → Trust Score Update → Decision → Forward/Block

Pipeline Stages

  1. Intent Analysis — Classifies the action's purpose (benign, malicious, suspicious)
  2. Prompt Injection Scan — Detects injection attempts in the prompt
  3. Risk Assessment — Calculates risk score (0-100) based on action parameters
  4. Policy Check — Validates against configured security policies
  5. Trust Score Update — Updates agent's trust score based on action
  6. Decision — ALLOW or BLOCK based on all checks

Trust Engine #

The Trust Engine maintains dynamic trust scores for each agent based on their behavior patterns:

Trust Score Impact
Trust scores range from 0-100. Low scores (<50) trigger additional scrutiny, while high scores (>80) allow faster processing. Scores are updated based on action outcomes and policy violations.

Trust Factors

  • Action History — Past actions and their outcomes
  • Policy Compliance — Adherence to configured policies
  • Time Patterns — Unusual timing or frequency
  • Resource Access — Sensitivity of accessed resources

SDK Integration #

Integrate Kavach AI into your application using our SDKs:

Python
from kavach_ai import KavachProxy

# Initialize the proxy
proxy = KavachProxy(
endpoint="https://api.kavach.ai/v1",
api_key="your-api-key"
)

# Wrap your MCP client
mcp_client = proxy.wrap(mcp_client)

# Tool calls are now intercepted
result = mcp_client.call_tool(
name="database_query",
args={"query": "SELECT * FROM users"}
)

# Check the decision
if result.decision == "ALLOW":
print(f"Action allowed. Risk score: {result.risk_score}")
else:
print(f"Action blocked. Reason: {result.reason}")
JavaScript
const { KavachProxy } = require('@kavach-ai/sdk');

// Initialize the proxy
const proxy = new KavachProxy({
endpoint: 'https://api.kavach.ai/v1',
apiKey: 'your-api-key'
});

// Wrap your MCP client
const mcpClient = proxy.wrap(mcpClient);

// Tool calls are now intercepted
const result = await mcpClient.callTool({
name: 'database_query',
args: { query: 'SELECT * FROM users' }
});

// Check the decision
if (result.decision === 'ALLOW') {
console.log(`Action allowed. Risk score: ${result.riskScore}`);
} else {
console.log(`Action blocked. Reason: ${result.reason}`);
}
TypeScript
import { KavachProxy, SecurityResult } from '@kavach-ai/sdk';

// Initialize the proxy
const proxy = new KavachProxy({

} else {
console.log(`Action blocked. Reason: ${result.reason}`);
}

MCP Integration #

Kavach AI provides native support for the Model Context Protocol (MCP). Configure your MCP server to route through Kavach AI:

MCP Configuration
# mcp_config.json
{
"proxy": {
"enabled": true,
"endpoint": "https://api.kavach.ai/v1",
"api_key": "your-api-key",
"timeout": 2000
},
"servers": [
{
"name": "database",
"enabled": true,
"policies": ["read-only"]
}
]
}

FAQ #

What is the latency impact of Kavach AI?
Kavach AI adds less than 2ms of latency to each action. Our optimized pipeline ensures minimal performance impact while providing comprehensive security analysis.
Can I use Kavach AI with any AI framework?
Yes, Kavach AI works with any AI framework that supports MCP or can be integrated via our REST API. We have specific SDKs for Python, JavaScript, TypeScript, and Go.
Is my data secure with Kavach AI?
Absolutely. Kavach AI is designed for enterprise security. You can self-host the entire stack, and all data remains within your infrastructure. We never store or transmit your data externally.
How do I get an API key?
Sign up at kavach-ai.dev to get your API key. The free tier includes 10,000 requests per month. Enterprise plans offer unlimited requests and dedicated support.

Troubleshooting #

Common Issues

Connection Timeout

If you're experiencing connection timeouts, check your network settings and ensure the API endpoint is accessible. Try increasing the timeout parameter in your configuration.

Invalid API Key

Verify your API key is correct and hasn't expired. Check your dashboard to regenerate the key if needed.

Policy Not Blocking Expected Actions

Review your policy rules to ensure they match the action patterns you want to block. Use the test endpoint to validate your policies.
